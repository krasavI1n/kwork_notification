# Contributing to Kwork Monitor

Спасибо за интерес к улучшению Kwork Monitor! Мы приветствуем любой вклад в проект.

## Как внести вклад

### Сообщить об ошибке

Если вы нашли баг:

1. Проверьте, что баг еще не [зарегистрирован](../../issues)
2. Создайте новый issue с описанием:
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Версия Python и ОС
   - Логи ошибок (если есть)

### Предложить улучшение

Есть идея для новой функции?

1. Создайте issue с тегом `enhancement`
2. Опишите:
   - Что вы хотите добавить
   - Зачем это нужно
   - Как это должно работать

### Отправить Pull Request

1. **Форкните репозиторий**

2. **Создайте feature branch**
   ```bash
   git checkout -b feature/amazing-feature
   ```

3. **Внесите изменения**
   - Следуйте стилю кода проекта
   - Добавьте комментарии где необходимо
   - Не включайте лишние файлы

4. **Протестируйте**
   - Запустите приложение
   - Проверьте основную функциональность
   - Убедитесь, что ничего не сломалось

5. **Закоммитьте**
   ```bash
   git commit -m "Add: описание изменений"
   ```
   
   Формат commit message:
   - `Add: новая функция`
   - `Fix: исправление бага`
   - `Update: обновление существующего`
   - `Refactor: рефакторинг кода`
   - `Docs: обновление документации`

6. **Запушьте branch**
   ```bash
   git push origin feature/amazing-feature
   ```

7. **Откройте Pull Request**
   - Опишите что изменили
   - Укажите связанные issues (если есть)
   - Приложите скриншоты (для UI изменений)

## Стиль кода

- Используйте **PEP 8** для Python кода
- Имена переменных и функций: `snake_case`
- Имена классов: `PascalCase`
- Константы: `UPPER_CASE`
- Docstrings для классов и публичных методов
- Комментарии на русском или английском
