# Kwork Project Monitor

Десктопное приложение для мониторинга новых проектов на бирже Kwork.

## Возможности

- 🔍 Автоматический мониторинг новых проектов по указанному URL
- 🔔 Desktop уведомления при появлении новых проектов
- 🔊 Звуковые алерты
- ⚙️ Настраиваемый интервал проверки (15-120 секунд)
- 💾 Автоматическое отслеживание просмотренных проектов
- 🎨 Минималистичный интерфейс

## Установка

1. Убедитесь, что установлен Python 3.9+

2. Установите зависимости:
```bash
pip install -r requirements.txt
```

## Запуск

```bash
py src/main.py
```

Или на Linux/Mac:
```bash
python3 src/main.py
```

## Использование

### Шаг 1: Настройка фильтров на Kwork

1. Откройте в браузере [kwork.ru/projects](https://kwork.ru/projects)
2. Настройте нужные фильтры проектов:
   - Выберите категории
   - Установите бюджет (от/до)
   - Выберите тип проектов (новые/все)
   - Примените другие фильтры по необходимости
3. Нажмите кнопку применения фильтров или дождитесь обновления страницы
4. Скопируйте URL из адресной строки браузера
   - Пример: `https://kwork.ru/projects?kworks-filters%5B%5D=0&kworks-filters%5B%5D=1&a=1&page=1`

### Шаг 2: Запуск мониторинга

1. **Запустите приложение**:
   ```bash
   py src/main.py
   ```

2. **Вставьте скопированный URL** в текстовое поле

3. **Настройте параметры (⚙)** при необходимости:
   - Интервал проверки (15-120 сек) - рекомендуется 20-30 сек
   - Включить/отключить desktop уведомления
   - Включить/отключить звук
   - Время хранения отслеженных проектов (часов)

4. **Нажмите "▶ Start"** для запуска мониторинга

5. Приложение начнет проверять страницу с заданным интервалом

### Шаг 3: Получение уведомлений

Когда на бирже появятся новые проекты, соответствующие вашим фильтрам:
- Получите desktop уведомление с названием и ценой проекта
- Услышите звуковой сигнал
- Увидите проект в списке "Последние проекты (15)" в приложении
- Статус покажет "✓ Найдено новых проектов: N"

### Остановка мониторинга

- Нажмите **"⏸ Stop"** для временной остановки
- Закройте приложение для полного выхода
- При повторном запуске старые проекты не вызовут уведомлений (они сохранены в `data/state.json`)

## Структура проекта

```
kwork_notification/
├── src/
│   ├── main.py              # Точка входа
│   ├── ui/                  # Интерфейс PyQt5
│   ├── core/                # Логика мониторинга и парсинга
│   ├── services/            # Сервисы (уведомления, конфигурация)
│   └── models/              # Модели данных
├── data/
│   ├── config.json          # Пользовательские настройки
│   ├── state.json           # Отслеженные проекты
│   └── default_config.json  # Настройки по умолчанию
├── assets/
│   └── notification.wav     # Звук уведомления
└── requirements.txt
```

## Конфигурация

Настройки хранятся в `data/config.json`:

```json
{
  "url": "https://kwork.ru/projects?...",
  "monitoring_interval": 20,
  "notification_enabled": true,
  "sound_enabled": true,
  "retention_hours": 24
}
```

### Custom Headers

Для настройки кастомных HTTP заголовков отредактируйте `data/config.json`:

```json
{
  "custom_headers": {
    "User-Agent": "Mozilla/5.0...",
    "Cookie": "session=..."
  }
}
```

### CSS Селекторы

Для адаптации под изменения структуры Kwork можно настроить селекторы:

```json
{
  "selectors": {
    "card_container": ".want-card",
    "title_link": ".wants-card__header-title a",
    "price": ".wants-card__price",
    "description": ".wants-card__description-text"
  }
}
```

## Обработка ошибок

- **Ошибка соединения**: Автоматический retry с exponential backoff
- **Rate limiting (429)**: Автоматическое увеличение интервала
- **Ошибки парсинга**: Логируются, но мониторинг продолжается
- **Невалидный URL**: Проверка перед запуском

## Требования

- Python 3.9+
- PyQt5
- requests
- beautifulsoup4
- lxml
- win10toast (только Windows)

## Особенности

- Автоматическая очистка старых ID проектов (настраиваемо)
- Минимум 50 последних проектов всегда сохраняются
- Rate limiting уведомлений (не чаще 1 раз в 3 секунды)
- Graceful shutdown при закрытии приложения

## Устранение неполадок

**Не работают desktop уведомления:**
- Проверьте разрешения уведомлений Windows
- Убедитесь, что установлен `win10toast`

**Не находит проекты:**
- Проверьте URL (должен открываться в браузере)
- Проверьте CSS селекторы в настройках (если структура Kwork изменилась)
- Откройте `data/config.json` и проверьте `selectors`

**Python команда не найдена:**
- Используйте `py` вместо `python` на Windows
- Или укажите полный путь к Python
