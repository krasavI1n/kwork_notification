# 🚀 Kwork Project Monitor

[![Python](https://img.shields.io/badge/Python-3.9%2B-blue.svg)](https://www.python.org/downloads/)
[![PyQt5](https://img.shields.io/badge/PyQt5-5.15%2B-green.svg)](https://pypi.org/project/PyQt5/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

Автоматический мониторинг новых проектов на бирже фриланса [Kwork.ru](https://kwork.ru). Получайте мгновенные уведомления о появлении новых заказов, соответствующих вашим фильтрам.

![Kwork Monitor Screenshot](https://via.placeholder.com/800x500/f5f5f5/333333?text=Kwork+Monitor+Interface)

## ✨ Возможности

- 🔍 **Автоматический мониторинг** — проверка новых проектов с настраиваемым интервалом
- 🔔 **Desktop уведомления** — мгновенные Windows уведомления о новых проектах
- 🔊 **Звуковые алерты** — звуковое оповещение при появлении заказов
- ⚙️ **Гибкая настройка** — интервал проверки от 15 до 120 секунд
- 💾 **Умное отслеживание** — автоматическое запоминание просмотренных проектов
- 🎨 **Минималистичный интерфейс** — без лишних элементов, только функциональность
- 🛡️ **Обработка ошибок** — автоматические retry при сбоях сети
- 🔧 **Настраиваемые фильтры** — работает с любыми фильтрами Kwork

## 📦 Установка

### Требования

- Python 3.9 или выше
- Windows 10/11 (для desktop уведомлений)

### Шаги установки

1. **Клонируйте репозиторий**
```bash
git clone https://github.com/yourusername/kwork_notification.git
cd kwork_notification
```

2. **Установите зависимости**
```bash
pip install -r requirements.txt
```

3. **Запустите приложение**

Windows:
```bash
py src/main.py
```

Linux/Mac:
```bash
python3 src/main.py
```

## 🎯 Использование

### Быстрый старт

1. **Настройте фильтры на Kwork**
   - Откройте [kwork.ru/projects](https://kwork.ru/projects)
   - Выберите нужные категории, бюджет, тип проектов
   - Скопируйте URL из адресной строки

2. **Запустите мониторинг**
   - Вставьте URL в приложение
   - Настройте интервал проверки (рекомендуется 20-30 сек)
   - Нажмите **"▶ Start"**

3. **Получайте уведомления**
   - Desktop уведомление с названием и ценой
   - Звуковой сигнал
   - Проект появится в списке "Последние проекты"

### Детальная инструкция

#### Шаг 1: Настройка фильтров на Kwork

1. Откройте в браузере [kwork.ru/projects](https://kwork.ru/projects)
2. Настройте нужные фильтры проектов:
   - Выберите категории
   - Установите бюджет (от/до)
   - Выберите тип проектов (новые/все)
   - Примените другие фильтры по необходимости
3. Нажмите кнопку применения фильтров или дождитесь обновления страницы
4. **Скопируйте URL** из адресной строки браузера
   
   Пример URL:
   ```
   https://kwork.ru/projects?kworks-filters%5B%5D=0&kworks-filters%5B%5D=1&a=1&page=1
   ```

#### Шаг 2: Запуск мониторинга

1. **Запустите приложение**:
   ```bash
   py src/main.py
   ```

2. **Вставьте скопированный URL** в текстовое поле

3. **Настройте параметры** (кнопка ⚙) при необходимости:
   - **Интервал проверки**: 15-120 сек (рекомендуется 20-30 сек)
   - **Desktop уведомления**: включить/отключить
   - **Звуковые алерты**: включить/отключить
   - **Хранение проектов**: сколько часов помнить просмотренные

4. **Нажмите "▶ Start"** для запуска мониторинга

#### Шаг 3: Получение уведомлений

Когда на бирже появятся новые проекты:
- ✅ Desktop уведомление с названием и ценой проекта
- 🔊 Звуковой сигнал
- 📋 Проект появится в списке "Последние проекты (15)"
- 📊 Статус покажет "✓ Найдено новых проектов: N"

#### Остановка мониторинга

- **Пауза**: Нажмите **"⏸ Stop"** для временной остановки
- **Выход**: Закройте приложение
- **Важно**: При повторном запуске старые проекты не вызовут уведомлений (сохранены в `data/state.json`)

## ⚙️ Расширенная конфигурация

### Файл конфигурации

Все настройки хранятся в `data/config.json`. Можно редактировать вручную для продвинутых настроек:

```json
{
  "url": "https://kwork.ru/projects?...",
  "monitoring_interval": 20,
  "notification_enabled": true,
  "sound_enabled": true,
  "retention_hours": 24,
  "max_retries": 3,
  "timeout": 10
}
```

### Custom HTTP Headers

Для настройки кастомных заголовков (если требуется авторизация):

```json
{
  "custom_headers": {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
    "Cookie": "session=your_session_token",
    "Accept-Language": "ru-RU,ru;q=0.9"
  }
}
```

### CSS Селекторы

При изменении структуры сайта Kwork можно настроить селекторы:

```json
{
  "selectors": {
    "card_container": ".want-card",
    "title_link": ".wants-card__header-title a",
    "price": ".wants-card__price",
    "description": ".wants-card__description-text"
  }
}
```

### Технологический стек

- **UI Framework**: PyQt5 — нативный кроссплатформенный интерфейс
- **HTTP Client**: requests — надежные HTTP запросы с retry
- **HTML Parser**: BeautifulSoup4 + lxml — быстрый парсинг HTML
- **Notifications**: win10toast — нативные Windows уведомления
- **Sound**: winsound — встроенная библиотека для звука

## 🛠️ Устранение неполадок

### Desktop уведомления не работают

**Проблема**: Уведомления не появляются

**Решение**:
1. Проверьте разрешения уведомлений в Windows:
   - Настройки → Система → Уведомления и действия
   - Убедитесь, что уведомления включены
2. Проверьте установку `win10toast`:
   ```bash
   pip install --upgrade win10toast
   ```
3. Проверьте настройки в приложении (⚙ → Desktop уведомления)

### Проекты не обнаруживаются

**Проблема**: Мониторинг работает, но новые проекты не находятся

**Решение**:
1. **Проверьте URL**: откройте его в браузере — проекты должны отображаться
2. **Проверьте CSS селекторы**: структура Kwork могла измениться
   - Откройте `data/config.json`
   - Проверьте секцию `selectors`
   - При необходимости обновите селекторы (см. раздел "CSS Селекторы")
3. **Проверьте сеть**: убедитесь, что нет блокировки запросов

### Python команда не найдена

**Проблема**: `python` не распознается в командной строке

**Решение**:
- **Windows**: используйте `py` вместо `python`
  ```bash
  py src/main.py
  ```
- **Linux/Mac**: используйте `python3`
  ```bash
  python3 src/main.py
  ```

### Ошибка импорта модулей

**Проблема**: `ModuleNotFoundError: No module named 'PyQt5'`

**Решение**:
```bash
pip install -r requirements.txt
```

### Приложение зависает

**Проблема**: Интерфейс не отвечает

**Решение**:
1. Увеличьте интервал проверки (⚙ → 30-60 сек)
2. Проверьте интернет-соединение
3. Перезапустите приложение
